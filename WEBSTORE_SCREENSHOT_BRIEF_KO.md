# Chrome Web Store 스크린샷 브리프 (Korean)

## 목표
- 스토어 페이지에 들어왔을 때 5초 안에 아래 메시지가 전달되게 합니다.
  `Taillog는 콘솔 로그와 네트워크 요청을 더 깔끔하게 보여주는 프론트엔드 디버깅 패널이다.`

## 비주얼 방향
- 새로 정리한 미니멀 다크 UI를 사용합니다.
- 이전처럼 캐릭터 중심 빈 상태 화면은 전면에 내세우지 않습니다.
- 짧은 헤드라인과 높은 대비로 메시지를 빠르게 읽히게 만듭니다.
- 정보 밀도, 가독성, 집중도를 보여주는 컷을 우선합니다.
- 한 장에 모든 기능을 넣으려고 하지 않습니다.

## 스크린샷 구성안

### 스크린샷 1
- 헤드라인: `콘솔과 네트워크를 한 화면에서`
- 보여줄 것: `Network` 탭이 열린 메인 화면
- 목적: 제품이 무엇인지 첫 장에서 바로 설명
- 구성: 상단 내비게이션, 라이브 상태, 필터, 읽기 쉬운 요청 리스트 5~8개

### 스크린샷 2
- 헤드라인: `요청을 더 빠르게 스캔`
- 보여줄 것: 메서드 칩, 상태 코드, 검색창, 확장된 요청 상세가 함께 보이는 화면
- 목적: Network 화면의 실사용 가치를 증명
- 구성: 리스트가 중심이고, Payload/Response는 한 항목만 확장

### 스크린샷 3
- 헤드라인: `에러와 로그를 더 가볍게 확인`
- 보여줄 것: 여러 레벨 로그와 확장된 구조화 로그가 있는 `Console` 화면
- 목적: 두 번째 핵심 기능이 Console이라는 점을 보여줌
- 구성: 필터 세그먼트와 최근 로그 여러 개가 함께 보이게 구성

### 스크린샷 4
- 헤드라인: `자주 복사하는 텍스트를 가까이에`
- 보여줄 것: 3~4개의 클립보드 기록이 정리된 `Stash` 화면
- 목적: 보조 유틸리티 기능으로서의 가치를 전달
- 구성: 출처 배지, 시간, 짧은 본문 미리보기, 복사 액션 포함

### 스크린샷 5
- 헤드라인: `React 컴포넌트에서 바로 소스로`
- 보여줄 것: 단순화된 `Locator` 설정 화면
- 목적: React 개발자에게 유용한 부가 기능임을 전달
- 구성: 상태 토글, IDE 선택, 짧은 사용 가이드

## 카피 작성 원칙
- 헤드라인은 가능하면 8단어 이하로 짧게 유지합니다.
- 스크린샷 한 장당 메시지는 하나만 전달합니다.
- 기능명보다 사용 흐름 중심 언어를 씁니다.
- 좋은 예: `요청을 더 빠르게 스캔`
- 약한 예: `GET POST 필터와 복사 버튼이 있는 Network 탭`

## 캡처 체크리스트
- 빈 화면이 아니라 실제처럼 보이는 데이터로 캡처합니다.
- 너무 긴 URL만 잔뜩 보이는 장면은 피합니다.
- 시간, 상태 코드, 필터 상태가 보이게 합니다.
- 한 장에 확장된 상세는 1개만 넣습니다.
- 가능하면 별도 창 모드 크기로 캡처해 프레이밍을 안정적으로 맞춥니다.

## 추천 업로드 순서
1. Network 전체 화면
2. Network 상세 화면
3. Console 사용 화면
4. Stash 유틸리티 화면
5. Locator 유틸리티 화면
