# Chrome Web Store 스크린샷 브리프 (Korean)

## 목표

몇 초 안에 아래 메시지가 전달되도록 합니다.

`Taillog는 콘솔 로그와 네트워크 요청을 더 깔끔하게 보여주는 프론트엔드 디버깅 패널이다.`

## 비주얼 방향

- 현재의 미니멀 다크 UI를 사용합니다
- 빈 화면보다 실제 데이터가 있는 장면을 우선합니다
- 문구는 짧고 제품 중심으로 유지합니다
- 넓은 컷은 별도 창 모드로 캡처합니다

## 스크린샷 구성안

### 스크린샷 1
- 헤드라인: `콘솔과 네트워크를 한 패널에서`
- 보여줄 것: `Network`가 열린 메인 화면
- 포함 요소: 상단 앱 헤더, 연결 상태, 요약 칩, 필터, 요청 리스트
- 목적: 제품이 무엇인지 첫 장에서 바로 설명

### 스크린샷 2
- 헤드라인: `전체 DevTools 없이 요청 확인`
- 보여줄 것: 하나의 요청이 펼쳐진 `Network` 화면
- 포함 요소: 메서드 필터, 검색창, payload/response 상세
- 목적: 핵심 사용 흐름을 자세히 보여주기

### 스크린샷 3
- 헤드라인: `로그를 더 빠르게 파악`
- 보여줄 것: 여러 레벨 로그가 보이는 `Console` 화면
- 포함 요소: 레벨 필터와 확장된 구조화 로그 1개
- 목적: 두 번째 핵심 기능 강조

### 스크린샷 4
- 헤드라인: `복사한 텍스트를 가까이에`
- 보여줄 것: 여러 항목이 저장된 `Stash` 화면
- 포함 요소: 출처 배지, 시간, 미리보기 텍스트, 복사 액션
- 목적: 보조 도구로서의 가치 전달

### 스크린샷 5
- 헤드라인: `컴포넌트에서 바로 소스로`
- 보여줄 것: `Locator` 화면
- 포함 요소: 토글, 에디터 선택, 안내 카드
- 목적: React 전용 유틸리티 강조

### 스크린샷 6
- 헤드라인: `언어와 탭 상태를 그대로 유지`
- 보여줄 것: 언어 스위처가 보이는 화면
- 포함 요소: EN/KO 전환, 마지막 탭 유지 상태
- 목적: 다국어 지원과 설정 유지 같은 완성도 전달

## 카피 원칙

- 스크린샷 한 장당 메시지는 하나만 전달합니다
- 기능명 나열보다 사용 흐름 중심 문구를 씁니다
- 헤드라인은 가능하면 짧게 유지합니다

## 캡처 체크리스트

- 실제처럼 보이는 로그와 요청 데이터를 사용합니다
- 한 장에 확장된 상세는 1개만 넣습니다
- 필터와 시간 정보가 보이게 합니다
- 별도 창 캡처 시 어색한 빈 공간이 없도록 조정합니다
- 넓은 화면 예시도 최소 1장 포함합니다

## 추천 업로드 순서

1. Network 전체 화면
2. Network 상세 화면
3. Console 사용 화면
4. Stash 화면
5. Locator 화면
6. 언어 전환 및 상태 유지 화면

## 그래픽 오버레이용 보조 문구

- `프론트엔드 팀을 위한 깔끔한 디버깅`
- `덜 열고, 더 빨리 찾기`
- `로그와 요청을 한 패널에서`
