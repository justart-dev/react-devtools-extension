### 짧은 설명 (132자 이하)
```text
콘솔 로그와 네트워크 요청을 한눈에 보는 깔끔한 패널. 프론트엔드 디버깅을 더 빠르게 만듭니다.
```

### 짧은 설명 대안
```text
복잡한 DevTools 대신, 콘솔과 네트워크를 빠르게 훑을 수 있는 프론트엔드 디버깅 패널.
```

```text
로그, 요청, 클립보드 기록, React 컴포넌트 탐색을 더 단순한 흐름으로 정리한 디버깅 도구.
```

### 상세 설명
```text
Taillog는 프론트엔드 개발자를 위한 가벼운 디버깅 패널입니다.

복잡한 DevTools 화면 대신, 자주 보는 신호를 더 빠르게 확인할 수 있도록 설계했습니다.

Taillog는 가장 중요한 두 가지 흐름을 먼저 보여줍니다.
- Console 로그
- Network 요청

페이지에서 일어나는 변화를 빠르게 훑고, 필요한 항목만 펼쳐서 자세히 확인할 수 있습니다.

Taillog를 쓰는 이유
- 첫 화면이 단순해서 바로 상황 파악이 쉽습니다
- 최근 로그와 요청을 더 빠르게 스캔할 수 있습니다
- Payload와 Response를 간단하게 확인할 수 있습니다
- 보조 기능은 유지하되 메인 흐름을 방해하지 않습니다
- 모든 데이터는 로컬에만 저장됩니다

주요 기능

Console
- console.log, info, warn, error를 실시간으로 확인
- 레벨별 필터로 필요한 로그만 빠르게 확인
- 필요할 때만 펼쳐서 전체 payload 확인
- JSON과 구조화된 로그를 더 읽기 쉽게 표시

Network
- fetch와 XMLHttpRequest 요청을 실시간으로 확인
- 메서드별 필터와 URL/경로 검색 지원
- Payload와 Response를 한 번에 펼쳐서 확인
- 요청/응답 데이터를 빠르게 복사해 공유 가능

Stash
- 복사한 텍스트를 자동으로 저장
- 자주 쓰는 디버깅용 문자열을 빠르게 다시 복사
- 최근 클립보드 기록을 가볍게 관리

Locator
- Alt 또는 Option 키로 React 컴포넌트를 탐색
- 메타데이터가 있으면 선호 IDE에서 소스 파일 열기
- 개발 환경에서 사용할 때 가장 유용

프론트엔드 디버깅에 최적화
Taillog는 React 앱, API 호출이 많은 화면, 빠른 반복 개발 환경에서 특히 유용합니다. 전체 DevTools를 열기엔 부담스럽지만 로그와 요청은 바로 보고 싶을 때 잘 맞습니다.

개인정보 및 데이터
- 모든 데이터는 내 기기에만 저장됩니다
- 계정이 필요하지 않습니다
- 외부 서버로 동기화하지 않습니다

콘솔과 네트워크를 더 깔끔하게 확인하고 싶은 프론트엔드 개발자라면, Taillog는 그 흐름에 맞게 만들어진 도구입니다.
```

### 스토어 포지셔닝 메모
- 핵심 가치는 `Console + Network`를 먼저 보여주는 것입니다.
- `Stash`, `Locator`는 보조 기능으로 소개하고 메인 메시지를 흐리지 않게 유지합니다.
- Locator가 모든 환경에서 완전 무설정처럼 보이게 쓰는 표현은 피하는 편이 안전합니다.
- 기능 나열보다 실제 사용 흐름 중심의 문장으로 구성합니다.
